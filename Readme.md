## рҹ§  The Problem This Solves

Most data tools fail on large files. Upload a 1 GB CSV to a typical Flask app and it will crash вҖ” the server tries to hold the entire file in RAM.

**InsightFlow never does that.**

Every part of the pipeline вҖ” uploading, processing, visualizing, downloading вҖ” is engineered to work in small, memory-safe chunks. The full file never touches RAM. This makes it production-ready for real-world, large-scale datasets.

---

## вңЁ Features

| Category | Capabilities |
|---|---|
| **Upload** | Chunked upload via `Blob.slice()` вҖ” 5 MB slices, zero RAM usage |

| **Transformation** | Min-Max Scaling, Standardization, Label Encoding, One-Hot Encoding |
| **Dimensionality Reduction** | PCA, LDA, t-SNE, Feature Selection & Construction |
| **Visualization** | Histograms, Correlation Heatmaps (10Г—10), Scatter Plots (5,000-row sampled) |
| **Download** | Stream Parquet directly В· On-the-fly CSV conversion |
| **State Management** | SQLite-backed recipe persistence вҖ” survives browser refreshes |

## рҹҡҖ Getting Started

### Prerequisites

- Python **3.9 or higher**
- `pip` (comes with Python)
- A terminal / command prompt

---

### Step 1 вҖ” Clone the Repository

```bash
git clone https://github.com/YOUR_USERNAME/insightflow.git
cd insightflow
```

---

### Step 2 вҖ” Create a Virtual Environment

A virtual environment keeps InsightFlow's dependencies isolated from your system Python.

#### рҹҚҺ macOS / Linux

```bash
# Create the virtual environment
python3 -m venv venv

# Activate it
source venv/bin/activate

# Your terminal prompt will change to show (venv)
```

#### рҹӘҹ Windows (Command Prompt)

```cmd
:: Create the virtual environment
python -m venv venv

:: Activate it
venv\Scripts\activate.bat

:: Your terminal prompt will change to show (venv)
```

#### рҹӘҹ Windows (PowerShell)

```powershell
# Create the virtual environment
python -m venv venv

# Activate it
venv\Scripts\Activate.ps1

# If you get a permissions error, run this first:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

> рҹ’Ў **To deactivate** the virtual environment later, just type `deactivate` in any terminal.

---

### Step 3 вҖ” Install Dependencies

Make sure your virtual environment is **active** (you should see `(venv)` in your terminal), then run:

```bash
pip install -r requirements.txt
```

## рҹ”¬ How the Key Pipelines Work

<details>
<summary><b>рҹ“Ө Chunked Upload Pipeline</b></summary>

JavaScript slices the file into 5 MB pieces using `Blob.slice()`. Each piece is sent to Flask via `Fetch API`. Flask opens the file in **Append Binary (`ab`) mode** вҖ” raw bytes are written directly to disk without decoding into Python objects. RAM usage stays flat at ~0 MB regardless of file size. On the final chunk, the raw binary is converted to Parquet format.

</details>

<details>
<summary><b>рҹ§ҫ Recipe & State Pipeline</b></summary>

When you select a transformation (e.g., "Min-Max Scaling on column Salary"), no data is processed yet. The action is logged as a lightweight JSON object and pushed to SQLite. This means you can queue up 10 transformations, close the browser, reopen it, and your pending steps are still there вҖ” waiting to be executed.

</details>

<details>
<summary><b>вҡЎ Atomic Swap Pipeline</b></summary>

When you click **Save Changes**, Dask reads the recipe and builds a task graph. Processing happens partition-by-partition, writing results to a **temporary** directory. Only after 100% completion does it perform the Atomic Swap вҖ” deleting the old directory and instantly renaming the temp directory to the official path. If the server crashes mid-save, your original data is never touched.

</details>

<details>
<summary><b>рҹ“Ҡ Visualization Pipeline</b></summary>

For histograms, Dask computes frequency bins using `dask.array` вҖ” only counts, never raw rows, are sent to the browser. For scatter plots, Dask samples exactly **5,000 rows** across all partitions proportionally, making it safe to render regardless of dataset size. For heatmaps, Pearson correlation coefficients are computed and mapped to a Blue вҶ’ Red color scale.

</details>

<details>
<summary><b>рҹ“Ҙ Streaming Download Pipeline</b></summary>

**Parquet:** Flask streams the file directly from disk using `send_file`. **CSV:** The Parquet file is transcoded on-the-fly using `iter_batches()` вҖ” each tiny batch is converted to text and appended to the CSV output. The column headers are written exactly once (tracked with a `first_batch` boolean).

</details>

## рҹӣ пёҸ Troubleshooting

**`ModuleNotFoundError`** вҖ” Make sure your virtual environment is **activated** before running `pip install` or `python app.py`.

**`Permission denied` on Windows PowerShell** вҖ” Run `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` once, then reactivate the venv.

**Port already in use** вҖ” Change the port in `app.py`: `app.run(port=5001)` and visit `http://127.0.0.1:5001`.

**Slow CSV download** вҖ” Expected. CSV conversion from Parquet is done on-the-fly. For large files, Parquet download is significantly faster.
